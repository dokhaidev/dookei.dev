# This is NOT the Next.js you know

This version has breaking changes — APIs, conventions, and file structure may all differ from your training data. Read the relevant guide in `node_modules/next/dist/docs/` before writing any code. Heed deprecation notices.

# QUY TẮC NGÔN NGỮ
## 1. NGÔN NGỮ MẶC ĐỊNH
- Luôn trả lời bằng tiếng Việt, trừ khi user yêu cầu rõ ràng khác.
- Dù user hỏi bằng tiếng Anh, vẫn trả lời bằng tiếng Việt.
- Chỉ chuyển sang tiếng Anh khi user nói "use English", "answer in English", hoặc tương tự.

## 2. XỬ LÝ CODE VÀ KỸ THUẬT
- Code comments, console.log, log messages: tiếng Việt.
- Code variable names, function names: tiếng Anh (theo chuẩn dev).
- API response, error messages: tiếng Anh.
- Documentation trong code: tiếng Việt.

## 3. FORMAT TIẾNG VIỆT
- Dùng dấu tiếng Việt đầy đủ.
- Không dùng viết tắt quá nhiều.
- Giữ câu văn tự nhiên, dễ hiểu.

## 4. XỬ LÝ YÊU CẦU ĐA NGÔN NGỮ
- Nếu user hỏi về code cần tiếng Anh (React, Node.js docs): giải thích tiếng Việt, code tiếng Anh.
- Nếu user yêu cầu "explain in Vietnamese": ưu tiên tiếng Việt.

## 5. MẸO NHỎ
- Khi user copy code từ web tiếng Anh: dịch comment/explanation sang tiếng Việt.
- Trong commit message: dùng tiếng Việt ngắn gọn.

## 6. ƯU TIÊN
- Tiếng Việt > Tiếng Anh
- Tự nhiên > máy móc
- Ngắn gọn > dài dòng

Vui lòng luôn tuân thủ quy tắc này trong mọi phản hồi.
